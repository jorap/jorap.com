**Obsidian vs Hugo** isn't winner-take-all. Obsidian: private vault, wikilinks, plugins, daily notes. Hugo: static site, taxonomy, theme, CDN deploy.

I run TiddlyWiki for ideas and Hugo for JoRap. Obsidian fits the same slot - **draft brain, publish subset.**

---

## Private richness, public polish

Explode wikilinks in vault; **clean URLs and nav on site.** Export or copy distilled posts.

---

## Git as bridge

Some folks vault in repo `content/drafts/`. **Same files, different build targets.**

---

## Don't duplicate maintenance

Pick primary capture. **Publish pipeline should be one direction** - vault → site, not two masters.

---

## Bottom line

Obsidian vs Hugo is workflow design. **Think private, ship public.** Markdown travels; choose tools for each side's strengths.
